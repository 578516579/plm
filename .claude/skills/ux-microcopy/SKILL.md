# ux-microcopy — UX 微文案 skill v0.1

## 1. 何时调用
- "微文案 / microcopy / 按钮 / 错误 / 空状态"
- ux-designer §2.3

## 2. 原则

| 场景 | 反例 | 正例 |
|---|---|---|
| 按钮 | "确认" / "OK" | "保存项目" / "提交需求" (动+名) |
| 错误 | "出错了" / "Error 500" | "项目名不能为空, 请填写" |
| 空状态 | "暂无数据" | "还没有项目, 点击 + 创建第一个" |
| 确认对话 | "确定要删除吗?" | "删除项目 "X" 及其 N 个子任务?" |
| placeholder | "请输入" | "如: AgriPLM v1.0" |

## 3. 错误提示模板

```
[发生了什么] + [为什么] + [怎么办]
```

例: "提交失败: 名称已存在 (`AgriPLM`)。建议改为 "AgriPLM v2" 或在 [项目列表] 查找已有项目。"

## 4. 输出模板
```markdown
## §3 微文案

| 位置 | 当前 | 改进 | 理由 |
|---|---|---|---|
| 提交按钮 | "确认" | "提交项目" | 动+名, 减歧义 |
| 名称必填错 | "name required" | "项目名不能为空, 请填写" | 中文 + 给指引 |
```
