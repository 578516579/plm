## 关键讨论与决议

### D1 — 项目编号规则
- **讨论**：PRD 中标注"待 Phase 02 决"。提了 3 个候选：a) DB 自增 ID 直显 b) `PRJ-YYYY-NNNN` 前缀+流水号 c) 业务自定义。
- **决议**：暂留 Phase 02 设计阶段决议；建议倾向 (b)，因为人类友好、可在邮件中提及。**责任人：技术 lead，截止 Phase 02 评审前**。

### D2 — 标签功能（P-008）
- **讨论**：是否本期做？
- **决议**：**不做**，延到 v0.2。理由：标签的模型选型（多对多 vs JSON）影响数据库设计，本期先把核心 CRUD 跑通。**写入 PRD §6 Q2**。

### D3 — 项目级权限隔离
- **讨论**：业务方提"A 项目 PM 看不到 B 项目"是否必须？
- **决议**：**v0.1 不做**。本期所有登录用户可见全部项目；v0.2 引入 manager_user_id 过滤。理由：当前团队规模 < 10 人，权限隔离的边际价值低。**已写入 PRD §1.3 不做的事**。

### D4 — 模块工作流落位
- **讨论**：Project 模块代码放 `plm-system/system/business/project/` 还是新建独立模块 `plm-business`？
- **决议**：放 `plm-system` 子包（轻量方案）。理由：v0.1 单实体，没必要为一张表加 Maven 模块。后续 Phase / Task / Requirement / Change 多了再考虑拆模块（Phase 02 ADR 时复议）。
