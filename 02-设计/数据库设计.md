# 数据库设计

定义数据模型、表结构、索引策略、迁移规则。

## 1. ER 图
> 实体关系图（推荐 mermaid `erDiagram` 或飞书画板链接）。

## 2. 表清单
> 业务表（按模块分组）+ 系统表（sys_*, qrtz_*）；每张表注明 owner / 主键 / 关键索引。

## 3. 字段命名规范
> 通用列：`id, create_by, create_time, update_by, update_time, remark, del_flag`。
> 外键：`<table>_id`；逻辑删除：`del_flag` (`0=正常 / 2=删除`)。

## 4. 索引策略
> 哪些列必须加索引、组合索引顺序、避免回表的考虑。

## 5. 迁移与版本
> SQL 脚本存放：[plm-backend/sql/](../plm-backend/sql/)。新业务表用 `business-<entity>.sql` 命名。
> 后续如引入 Flyway / Liquibase 在此说明。
