- `plm-sprint` 需要查 task 计数（删除前置 / 健康度统计）

- `plm-task` 的 `TaskQueryServiceImpl` 实现该接口（Spring 自动注入）
- 编译期 `plm-sprint` ⊥ `plm-task`，运行期 Spring 拼接

## 2. 启动前端 monorepo

### 2.1 切换包管理器到 pnpm

```bash
# 一次性安装
npm install -g pnpm

cd plm-frontend
# 删除老 node_modules 避免冲突
rm -rf node_modules package-lock.json

# pnpm install 会同时安装根包 + packages/* 所有包
pnpm install

# 开发模式
pnpm run dev

# 期望: 主壳启动 → http://localhost
```

### 2.2 渐进式启用 @plm/* 路径

**Step 1**: 在新代码里改用 `@plm/{module}` 路径

```ts
// 旧写法 (仍可用)
import { listProject } from '@/api/business/project'

// 新写法 (推荐)
import { listProject } from '@plm/project/api'
import projectRoutes from '@plm/project/router'
import type { ProjectForm } from '@plm/project/types'
```

**Step 2**: 把 `src/router/index.ts` 中的 4 个业务路由替换为 packages 导入

```ts
import projectRoutes from '@plm/project/router'
import requirementRoutes from '@plm/requirement/router'
import sprintRoutes from '@plm/sprint/router'
import taskRoutes from '@plm/task/router'
import dashboardRoutes from '@plm/dashboard/router'
// ... 其余 25 stub 模块

const businessRoutes = [
  ...projectRoutes,
  ...requirementRoutes,
  ...sprintRoutes,
  ...taskRoutes,
  // stub 模块（渲染"建设中"占位页）
  ...dashboardRoutes,
  ...
]
```

**Step 3**: 删除 `src/{api,types,views}/business/` 老路径（确认主壳没引用后）

## 5. 常见问题

### Q1: mvn install 报循环依赖

Maven 编译期不允许 A→B & B→A。把反向调用抽成 `plm-common/api/IXxxQueryService.java` 接口，被调用方实现 + Spring @Service 注册，调用方 @Autowired 接口注入。参考 `ITaskQueryService` 模板。

### Q2: 新模块的 Mapper XML 没被扫描到

检查 `application.yml`：
```yaml
mybatis:
  typeAliasesPackage: cn.com.bosssfot.dv.plm.**.domain    # 通配 ** 已覆盖
  mapperLocations: classpath*:mapper/**/*Mapper.xml        # 通配 ** 已覆盖
```
和 `plm-framework/.../ApplicationConfig.java`:
```java
@MapperScan("cn.com.bosssfot.dv.plm.**.mapper")             // 已通配
```
新模块只需把 XML 放在 `src/main/resources/mapper/<module>/*.xml` 即可自动加载。

### Q3: 启动后端发现 stub 模块的菜单/字典/表都没建

stub 模块**只有 Java 空壳**，没 SQL。要真正使用 stub 需先实现 §3 Phase 02 + Phase 03。

### Q4: pnpm install 报 lockfile 错

```bash
rm -rf node_modules package-lock.json pnpm-lock.yaml
pnpm install --shamefully-hoist
```
