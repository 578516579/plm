# PLM 模块拆分架构（v0.3 重组方案）

| 字段 | 值 |
|---|---|
| 版本 | v1.0 |
| 触发原因 | 用户决策：把 `plm-system/business/*` monolith 拆为 31 个独立 Maven + pnpm 包，便于团队/AI 并行开发 |
| 范围 | AgriPLM 30 模块 + Sprint 单列 = 31 个业务模块 |
| 后端拆分策略 | 每业务模块独立 Maven jar |
| 前端拆分策略 | pnpm monorepo + 单 SPA 路由懒加载 |
| 迁移节奏 | Phase A：4 个成熟模块迁移示范 → Phase B：27 空壳拉起 → Phase C：前端 pnpm 改造 |

## 2. 后端 Maven 拓扑

```
plm/                                   (root pom, packaging=pom)
├── plm-common/                        [基础: BaseEntity, BaseController, utils]
├── plm-system/                        [现有: sys_* 用户/角色/菜单等]
├── plm-framework/                     [现有: Spring Security + JWT + AOP]
├── plm-quartz/                        [现有: 任务调度]
├── plm-generator/                     [现有: 代码生成器]
├── plm-admin/                         [启动入口 + 聚合 controller 扫描]
├── plm-project/                       [新业务 1]
├── plm-requirement/                   [新业务 2]
├── plm-sprint/                        [新业务 3]
├── plm-task/                          [新业务 4]
└── ... (其他 27 个业务模块)
```

### 2.1 业务模块通用结构

```
plm-{module}/
├── pom.xml
└── src/main/
    ├── java/cn/com/bosssfot/dv/plm/{module}/
    │   ├── domain/      {Entity}.java
    │   ├── mapper/      {Entity}Mapper.java
    │   ├── service/     I{Entity}Service.java
    │   ├── service/impl/{Entity}ServiceImpl.java
    │   └── controller/  {Entity}Controller.java
    └── resources/
        └── mapper/{module}/{Entity}Mapper.xml
```

### 2.2 依赖图

```
plm-admin
   ↓ depends on
plm-{31 业务模块}
   ↓ all depend on
plm-framework  →  plm-system  →  plm-common
```

**跨业务模块依赖（明示）**：
- `plm-task` depends on `plm-project` + `plm-requirement` + `plm-sprint`（FK 校验）
- `plm-sprint` depends on `plm-project`（FK + 删除前置检查 Task 时反向调用走 `plm-task` 但 sprint 不 depends task — 用 `@Lazy + ApplicationContext.getBean` 或事件总线解耦）
- `plm-requirement` depends on `plm-project`
- 其他模块按业务关系定（v0.3+ 实现时再定）

### 2.3 Mapper 扫描

`plm-admin/.../PlmApplication.java`：

```java
@SpringBootApplication
@MapperScan({
    "cn.com.bosssfot.dv.plm.system.*.mapper",   // sys_*
    "cn.com.bosssfot.dv.plm.*.mapper"           // 业务模块（通配）
})
public class PlmApplication { ... }
```

`mybatis-config` typeAliasesPackage 也用通配：

```yaml
mybatis:
  typeAliasesPackage: cn.com.bosssfot.dv.plm.**.domain
  mapperLocations: classpath*:mapper/**/*Mapper.xml
```

### 3.1 主壳路由聚合

`plm-shell/src/router/index.ts`：

```ts
import { createRouter } from 'vue-router'
const modules = import.meta.glob('../../plm-*/src/router.ts')
const routes = await Promise.all(
  Object.values(modules).map(load => load().then(m => m.default))
).then(r => r.flat())
export default createRouter({ routes: [...baseRoutes, ...routes] })
```

或显式 import（推荐，构建期可分析）：

```ts
import projectRoutes from '@plm/project/router'
import requirementRoutes from '@plm/requirement/router'
// ... 31 个
const routes = [...projectRoutes, ...requirementRoutes, ...]
```

## 5. 关键风险与缓解

| 风险 | 缓解 |
|---|---|
| 跨模块 Mapper XML 扫描路径错位 | `mapperLocations: classpath*:mapper/**/*Mapper.xml` 通配 |
| 跨模块 FK 校验循环依赖 | `plm-sprint` ↔ `plm-task` 是潜在循环；现行 `@Lazy` 注入已避免，新结构沿用 |
| 31 模块打包慢 | `mvn install -T 4 -pl <changed-module> -am` 增量构建 |
| 前端 monorepo 改动大 | 先把 monorepo 跑通 4 模块，27 个 stub 先空包不引入实际依赖 |
| 已运行后端进程 jar 锁 | 操作前先 taskkill /F /PID xxx |
| 中文乱码再现 | 启动脚本必须带 `-Dfile.encoding=UTF-8`；CI 跑 `check-encoding-runtime.sh` |
