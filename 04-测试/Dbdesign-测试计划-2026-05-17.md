## 1. 测试范围
- 单元测试 (Service 层): mvn -pl plm-dbdesign test
- E2E 测试 (前端 + 后端集成): npm run test:e2e -g "Dbdesign"
- 状态机覆盖: 见 [PRD-MAPPING.md §3](../PRD-MAPPING.md) `dbdesign` 行,合法 + 非法转换全覆盖

- [Dbdesign-functional.md](测试用例库/Dbdesign-functional.md) — 功能用例 (用户视角:做什么 + 预期),覆盖 CRUD 主流程 + 状态机推进 + 必填校验

## 5. 风险

| 等级 | 风险 | 缓解 |
|---|---|---|
| P0 | AI 调用失败或超时 (OpenAI / DeepSeek 接口 5xx / 限流) 导致 Dbdesign 生成流程中断 | Mock AI 返回做单测 + Service 层 3 次指数退避重试 + 失败降级到草稿状态 |
| P0 | AI 幻觉:返回字段值不在字典白名单 (如 status / priority) 触发 604 | Service 层字典白名单二次校验 + AI 输出落库前 Schema 校验 + 人工 review 入口 |
| P1 | 状态机非法转换 (草稿 → 已发布跳过中间审核) | Service 单测覆盖合法+非法转换矩阵 + 后端 transitTo() 白名单 + 返 601 |
| P1 | 多人并发编辑同一 Dbdesign 实例导致脏写 | update_time 乐观锁字段 + 后端 version 校验 + 返 409 Conflict |
| P2 | UI 兼容性 (AI 生成富文本在 Chrome / Edge 1366x768 视口渲染差异) | E2E 覆盖 Chrome+Edge + 视口断点 + 富文本 sanitize |
