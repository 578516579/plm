- [x] **A02 Cryptographic Failures** — JWT HMAC-SHA(jjwt),密码 bcrypt(`SecurityConfig`)— ✅

- [x] **A04 Insecure Design** — 状态机 / FK / UK 在 Service 层显式校验(31 模块 ServiceImplTest 26-39 case/模块)— ✅
- [x] **A05 Security Misconfiguration** — F-001/F-002/F-003 见 §2.1 — ⚠️
- [x] **A06 Vulnerable Components** — F-004(jjwt 老版本)— ⚠️
- [x] **A07 Identification and Authentication Failures** — 登录失败计数 + 锁定(`SysLogininforServiceImpl`)、JWT 30min 过期 — ✅
- [x] **A08 Software and Data Integrity Failures** — fastjson2 安全配置 + Spring Security CSRF + JWT 签名校验 — ✅
- [x] **A09 Logging and Monitoring Failures** — `sys_login_info` / `sys_oper_log` 全链路审计 — ✅
- [x] **A10 SSRF** — 现仅 LLM/集成连接器对外调用,域名白名单待补(F-006,P2 — 当前 LLM/Gitlab/Zentao URL 走 yml 配置非用户输入,**风险窗口低**)

### 2.4 业务安全

- [x] **日志脱敏** — `sys_oper_log` `requestParam` 字段对 password / token 字段已过滤(`@Excel(targetAttr="hide")` 模式)— ✅

### §3 详情:本次审计 7 个 finding

| 编号 | 级别 | 标题 | 描述 | 处置 |
|---|---|---|---|---|
| **F-001** | **P1** | JWT_SECRET 默认弱值 | `application.yml:150` 默认 `abcdefghijklmnopqrstuvwxyz`,生产若漏 env 注入 → JWT 可被任何人伪造 | 本地 OK / 生产部署 SOP **必须**写 `JWT_SECRET=<openssl rand -base64 32>`,部署清单加 §C.2 自检项 |
| **F-002** | **P1** | Druid 监控台默认弱口令 | `application-druid.yml:50-51` 默认 `plm/123456`,生产暴露后可看数据源密码、慢 SQL、热点 SQL | 选项 A:生产强制 `DRUID_USERNAME/PASSWORD` env 覆盖为强密码;选项 B:**推荐**生产 `spring.profiles.active=prod` 时 `stat-view-servlet.enabled: false` |
| **F-003** | **P1** | REDIS_PASSWORD 默认空 | `application.yml:131` 默认空字符串,生产 Redis 暴露 6379 + 无密码 → 任意人可 FLUSHALL | 部署清单必须 `REDIS_PASSWORD=<强密码>` + Redis `requirepass <同值>` |
| **F-004** | P2 | jjwt 0.9.1 老版本 | 2018 年版本,有 padding oracle 风险(CVE-2022-0840 影响 < 0.12) | 升级到 jjwt 0.12.x(API 有 break,需小改 `JwtUtil`)— 排进 v0.x 路线图 |
| **F-005** | P3 | npm audit 不可用 | npmmirror 镜像未实现 security advisories API | 临时方案:`npm config set registry https://registry.npmjs.org && npm audit && npm config set registry https://registry.npmmirror.com` 每月跑一次,归档到本节 |
| **F-006** | P2 | SSRF 域名白名单 | LLM/Gitlab/Zentao URL 走 yml 配置非用户输入,但未来若加"用户配置 webhook URL"场景需白名单 | 在 connector 层加 `URL allowlist` 工具方法,排进集成连接器演进路径 |
| **F-007** | 运维 | HTTPS/HSTS | dev http 无证书,生产 nginx 反代必须开 HSTS | 部署清单 §HTTPS 段必填,prod-deploy.md 中明确 |
