# 性能报告

每次压测的输入、过程、结果、瓶颈分析与改进建议。

## 文件命名

`YYYY-MM-DD_<模块>_v<版本号>.md`，例如 `2026-05-15_login_v0.1.0.md`

## 报告模板

```markdown
# 性能报告：<模块> v<版本号>

## 测试目标
> 验证 N QPS 下 P95 ≤ X ms / 错误率 ≤ Y%。

## 测试环境
- 后端机器规格 / 副本数
- DB 规格 / 数据量级
- Redis / 网络

## 压测工具与脚本
> JMeter / wrk / k6 / Gatling — 脚本链接

## 关键指标
| 场景 | QPS | P50 | P95 | P99 | 错误率 | CPU | 内存 |

## 瓶颈分析
> 数据库慢查询？连接池？GC？

## 改进建议
> 短期 / 长期。
```

## 长期归档

测试原始数据（CSV / HAR / JTL）建议存到对象存储，本目录只放摘要 + 链接。
