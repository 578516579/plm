## 7. 根因复诊 (v1.1, 2026-05-18)

### 7.1 静态分析方法

不启动服务,通过对比前端 .vue 文件中实际 label/按钮文案与 spec 期望,加 sys_menu seed SQL 验证路由注册:

| 验证项 | 方法 | 结果 |
|---|---|---|
| `getByLabel(/项目编号/)` | `Grep el-form-item label="项目编号"` in `packages/plm-project/src/views/index.vue` | ✅ 第 76 行存在 |
| `getByLabel(/项目名称/)` | 同上 | ✅ 第 81 行存在 |
| `getByRole('button', {name: /新增/})` | `Grep '新增' in packages/plm-requirement/src/views/index.vue` | ✅ 第 38 行存在 (`<el-button>新增</el-button>`) |
| `.el-table` | element-plus 标准组件,前端 `<el-table v-loading=...>` 必出 | ✅ 标签存在 |
| `goto('/business/project')` 路由可达 | `sql/business-*.sql` sys_menu seed 13 项 | ✅ 全部注册 (`business/project/index`, `business/requirement/index` 等) |
| packages monorepo 路由解析 | `src/store/modules/permission.ts:122-144` 双层 `import.meta.glob` 扫 `src/views/` 和 `packages/plm-*/src/views/` | ✅ 正确解析 |

### 7.2 真实根因 (高置信度)

5 fail 全部死在 spec 内 `await page.waitForSelector('.el-table', { timeout: 10_000 })` 或类似 10s 等待 — **10s timeout 不足以覆盖首次访问 packages/plm-* 模块的完整启动链路**:

```
goto('/business/project')
  → axios interceptor 读 Admin-Token cookie 注入 Authorization 头 (~10ms)
  → GET /api/getInfo (~50-200ms, 拉用户/角色/权限)
  → GET /api/getRouters (~50-200ms, 拉动态菜单 sys_menu)
  → permission.ts 遍历菜单 → router.addRoute() (~100ms)
  → 触发重定向到真实路由 /business/project
  → vite dev server 首次编译 packages/plm-project/src/views/index.vue (**冷启动 5-15s, vite 6 + TS + Element Plus**)
  → Vue 挂载 + el-table v-loading="true" 拉初始数据 (~500ms)
  → el-table 数据就绪 v-loading="false" → DOM 实际可见
```

vite 首次编译 packages monorepo 子包是 5-15 秒级别,**远超 spec 10s timeout**。后续访问会有 vite cache,通常 <1s,所以纯 API 层测试都过、UI 层每个模块第一次访问就 fail。

### 7.3 修复方向 (3 选 1, 不互斥)

| 方案 | 改动 | 优点 | 缺点 |
|---|---|---|---|
| **A. 提升 UI 测试 timeout 到 60s** | 5 spec 改 `{ timeout: 60_000 }` | 最快,1 行改 5 处 | 治标 — 单 case 慢 5-15s |
| **B. 全局 globalSetup 预热 vite** | 加 `playwright.config.ts` globalSetup,跑前先 `goto('/business/project')` 等 13 个模块路径预热 | 真正解决根因,后续 UI 测试 <1s | globalSetup 增 ~2 分钟启动 |
| **C. 改成 webServer 模式 + 编译预热** | playwright `webServer` 启动 vite,加 `reuseExistingServer` + 等 chunks 预编译 | 最专业 | 改动 config + vite 配置最大 |

**推荐**: 先做 **A** (快速验证),再做 **B** (治本)。
