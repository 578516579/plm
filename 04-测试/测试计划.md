# 测试计划

定义本期测试的范围、策略、资源、进度、风险。

## 1. 测试范围
> 哪些模块要测、哪些不测、依赖系统是 mock 还是真实联调。

## 2. 测试类型
- [ ] 单元测试（开发自测）
- [ ] 接口测试（Postman / RestAssured）
- [ ] 集成测试
- [ ] UI 自动化（如 Playwright）
- [ ] 性能测试（详见 [性能报告/](性能报告/)）
- [ ] 安全测试（详见 [安全审计.md](安全审计.md)）
- [ ] 回归测试

## 3. 测试环境
| 环境 | URL | 数据情况 | 维护人 |
|---|---|---|---|
| dev | http://localhost | 本地 | |
| staging | TBD | 脱敏生产数据子集 | |
| prod | TBD | 真实 | 仅 oncall |

## 4. 进入 / 退出标准
- 进入：代码冻结 + 用例覆盖率 ≥ X%
- 退出：P0/P1 bug 全部修复，P2 ≤ N 个

## 5. 风险
> 时间、人力、环境不稳定、依赖外部接口等。
