## 1. 发布范围
Dora 模块 v1.0 — 字段 + 状态机 + API + 字典 + 菜单全部就绪 (PRD-MAPPING §2 已对齐)。

## 2. 发布步骤
1. 数据库迁移: `mysql plm < sql/business-dora.sql`
2. 后端发布: `mvn -pl plm-dora install` + 重启 plm-admin
3. 前端发布: `npm run build` + 部署 plm-frontend
4. 烟雾测试: 通过菜单访问 + 创建 1 条数据 + 关键状态机演进
5. E2E 全套: `npm run test:e2e -g "Dora"`

## 3. 回滚预案
- DB: `mysql plm < sql/business-dora-rollback.sql`
- Code: git revert <feat commit>
- 菜单/字典:rollback SQL 自动清理

## 4. 监控

### 4.1 监控点 (3 维)

| 维度 | 指标 | 数据源 | 健康阈值 |
|---|---|---|---|
| **业务** | `tb_dora` 日新增行数 | MySQL count(create_time>=今日) | 小模块 0-50 / 日 |
| **业务** | DORA 4 指标采集失败数 | 应用日志 + Prometheus | ≤ 3 / 日 |
| **业务** | 当前 DORA 健康分 (LeadTime/DeployFreq/CFR/MTTR 综合) | 聚合任务输出 | ≥ 70 / 100 |
| **接口** | `/business/dora/*` 错误率 | Spring Actuator + Prometheus | 错误率 ≤ 1% |
| **接口** | `/business/dora/list` P95 响应时间 | APM | < 1000 ms |
| **接口** | `/business/dora/metrics` 报表 P95 | APM | < 3000 ms |
| **资源** | tb_dora 表 size 增长率 | MySQL information_schema | 月增 < 10% |

### 4.2 告警阈值

| 等级 | 触发条件 | 通知渠道 | 处理 SLA |
|---|---|---|---|
| P0 | 接口 5xx 持续 > 5 分钟 | PagerDuty + 飞书 | 30 分钟响应 |
| P1 | DORA 健康分 < 50 (持续 3 天) | 飞书 | 工作日响应 |
| P1 | DORA 采集失败数 > 5 / 小时 | 飞书 | 2 小时响应 |
| P1 | 列表查询 P95 > 3s 连续 10 分钟 | 飞书 | 2 小时响应 |
| P2 | 表行数月增 > 50% (异常增长) | 邮件 | 工作日响应 |

### 4.3 日志关键字

后端 Logback 输出, ELK / Loki 索引:
- `ERROR.*DoraService` — Service 层异常
- `WARN.*DoraController.*RejectedException` — 业务规则拒绝
- `INFO.*Dora.*metric-collect` — DORA 指标采集审计 (含 4 维度数据)
- `WARN.*Dora.*metric-collect-failed` — 指标采集失败
- `WARN.*Dora.*deadlock` — MySQL 死锁
