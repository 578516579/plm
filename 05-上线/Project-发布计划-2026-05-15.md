## 1. 范围

### 包含

- `tb_project` 业务表 + 5 个索引 + 2 个字典类型 + 8 条字典数据
- 7 条菜单数据（业务管理父菜单 2000 + 项目管理 2010 + 5 个按钮权限 2011-2015）
- admin 角色（role_id=1）授权所有 7 个菜单
- 后端：`ProjectController` + `ProjectService` + `ProjectMapper` + `Project` domain
- 前端：`/business/project` 路由 + 列表 + 新增/修改/删除/导出对话框
- 核心业务规则：
  - `generateProjectNo` 自动生成 `PRJ-YYYY-NNNN`（ADR-0001）
  - 状态机 5x5 转换矩阵 + 终态保护（PRD §3.3）
  - 字段校验 + 错误码 601/602/604/701（API §3.2 / §3.3）

### 不包含（v0.2 backlog）

- 项目模板复制（"基于已有项目快速建新项目"）
- 甘特图可视化
- 项目级权限隔离（"A 项目 PM 看不到 B 项目"）
- 评论功能
- 标签功能（PRD Q2 决定 v0.2 做）
- 工时 / 财务对账接口

## 4. 灰度策略（按 early 简化）

`early / internal-tool / solo` → **单步全量**。

理由：
- 项目无外部用户
- 业务方 = 测试方 = 开发方（Wjl 一人）
- 仅有 admin 角色访问
- E2E 测试已覆盖 5 个关键场景

若未来：
- `stable` 阶段：灰度 5% → 50% → 100%
- `mature` 阶段：灰度 1% → 5% → 50% → 100%

## 7. 回滚预案

详见 [Runbook §"Project 模块回滚"](Runbook.md)。

关键命令：

```bash
# 1. 后端回滚（git tag 切到上一个）
git checkout plm-0.0.0 -- plm-backend/   # 假设上一版叫 0.0.0（首版无此 tag，所以本期回滚 = 撤销）

# 2. 数据库回滚
mysql -uroot -p... --default-character-set=utf8mb4 plm <<'SQL'
DELETE FROM sys_role_menu WHERE menu_id BETWEEN 2000 AND 2015;
DELETE FROM sys_menu WHERE menu_id BETWEEN 2000 AND 2015;
DELETE FROM sys_dict_data WHERE dict_type IN ('biz_project_type', 'biz_project_status');
DELETE FROM sys_dict_type WHERE dict_type IN ('biz_project_type', 'biz_project_status');
DROP TABLE IF EXISTS tb_project;
SQL

# 3. 重启后端
java -jar plm-admin/target/plm-admin.jar --server.port=8081
```

> 因 v0.1.0 是首版，回滚 = 完全撤销（无上一版可切）。Runbook 已含完整撤销 SQL。
