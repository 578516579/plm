## 1. 发布范围
Release 模块 v1.0 — 字段 + 状态机 + API + 字典 + 菜单全部就绪 (PRD-MAPPING §2 已对齐)。

## 2. 发布步骤
1. 数据库迁移: `mysql plm < sql/business-release.sql`
2. 后端发布: `mvn -pl plm-release install` + 重启 plm-admin
3. 前端发布: `npm run build` + 部署 plm-frontend
4. 烟雾测试: 通过菜单访问 + 创建 1 条数据 + 关键状态机演进
5. E2E 全套: `npm run test:e2e -g "Release"`

## 3. 回滚预案
- DB: `mysql plm < sql/business-release-rollback.sql`
- Code: git revert <feat commit>
- 菜单/字典:rollback SQL 自动清理

## 4. 监控

### 4.1 监控点 (3 维)

| 维度 | 指标 | 数据源 | 健康阈值 |
|---|---|---|---|
| **业务** | `tb_release` 日新增行数 | MySQL count(create_time>=今日) | 小模块 0-30 / 日 |
| **业务** | 发布失败数 (status='failed') | MySQL count(status='failed') | ≤ 3 / 日 |
| **业务** | 回滚次数 (rolled_back) | MySQL count(status='rolled-back' AND last_action>=今日) | ≤ 2 / 日 |
| **接口** | `/business/release/*` 错误率 | Spring Actuator + Prometheus | 错误率 ≤ 1% |
| **接口** | `/business/release/list` P95 响应时间 | APM | < 1000 ms |
| **接口** | `/business/release/execute` P95 触发延迟 | APM | < 3000 ms |
| **资源** | tb_release 表 size 增长率 | MySQL information_schema | 月增 < 10% |

### 4.2 告警阈值

| 等级 | 触发条件 | 通知渠道 | 处理 SLA |
|---|---|---|---|
| P0 | 接口 5xx 持续 > 5 分钟 | PagerDuty + 飞书 | 30 分钟响应 |
| P0 | 发布失败数 > 3 / 小时 (持续故障) | PagerDuty + 飞书 | 30 分钟响应 |
| P0 | 单次发布回滚 (任意一次) | PagerDuty + 飞书 | 30 分钟响应 |
| P1 | 列表查询 P95 > 3s 连续 10 分钟 | 飞书 | 2 小时响应 |
| P2 | 表行数月增 > 50% (异常增长) | 邮件 | 工作日响应 |

### 4.3 日志关键字

后端 Logback 输出, ELK / Loki 索引:
- `ERROR.*ReleaseService` — Service 层异常
- `WARN.*ReleaseController.*RejectedException` — 业务规则拒绝 (如未通过 Gate 尝试发布)
- `INFO.*Release.*state-transit` — 状态机推进审计 (draft→approved→deploying→released→closed/rolled-back)
- `ERROR.*Release.*deploy-failed` — 部署失败 (含步骤 / 错误码)
- `WARN.*Release.*rollback` — 回滚审计 (含触发人 / 原因)
- `WARN.*Release.*deadlock` — MySQL 死锁
