## 1. 发布范围
Testplan 模块 v1.0 — 字段 + 状态机 + API + 字典 + 菜单全部就绪 (PRD-MAPPING §2 已对齐)。

## 2. 发布步骤
1. 数据库迁移: `mysql plm < sql/business-testplan.sql`
2. 后端发布: `mvn -pl plm-testplan install` + 重启 plm-admin
3. 前端发布: `npm run build` + 部署 plm-frontend
4. 烟雾测试: 通过菜单访问 + 创建 1 条数据 + 关键状态机演进
5. E2E 全套: `npm run test:e2e -g "Testplan"`

## 3. 回滚预案
- DB: `mysql plm < sql/business-testplan-rollback.sql`
- Code: git revert <feat commit>
- 菜单/字典:rollback SQL 自动清理

## 4. 监控

### 4.1 监控点 (3 维)

| 维度 | 指标 | 数据源 | 健康阈值 |
|---|---|---|---|
| **业务** | `tb_testplan` 日新增行数 | MySQL count(create_time>=今日) | 小模块 0-30 / 日 |
| **业务** | 测试计划评审未通过数 | MySQL count(status='rejected') | ≤ 3 |
| **业务** | 计划下用例覆盖率 (cases_planned/cases_required) | 测试计划聚合 | ≥ 80% |
| **接口** | `/business/testplan/*` 错误率 | Spring Actuator + Prometheus | 错误率 ≤ 1% |
| **接口** | `/business/testplan/list` P95 响应时间 | APM | < 1000 ms |
| **资源** | tb_testplan 表 size 增长率 | MySQL information_schema | 月增 < 10% |

### 4.2 告警阈值

| 等级 | 触发条件 | 通知渠道 | 处理 SLA |
|---|---|---|---|
| P0 | 接口 5xx 持续 > 5 分钟 | PagerDuty + 飞书 | 30 分钟响应 |
| P1 | 列表查询 P95 > 3s 连续 10 分钟 | 飞书 | 2 小时响应 |
| P1 | 计划用例覆盖率 < 50% (低质量计划) | 飞书 | 2 小时响应 |
| P2 | 表行数月增 > 50% (异常增长) | 邮件 | 工作日响应 |

### 4.3 日志关键字

后端 Logback 输出, ELK / Loki 索引:
- `ERROR.*TestplanService` — Service 层异常
- `WARN.*TestplanController.*RejectedException` — 业务规则拒绝
- `INFO.*Testplan.*state-transit` — 状态机推进审计 (draft→reviewing→approved→executing→closed)
- `WARN.*Testplan.*deadlock` — MySQL 死锁
