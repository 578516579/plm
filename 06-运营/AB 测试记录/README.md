# A/B 测试记录

> 目录名因文件系统不允许 `/`，落地为 `AB 测试记录`，语义仍是 **A/B Testing**。

每个 A/B 实验一份独立 .md，记录假设、分组、指标、结论。

## 文件命名

`YYYY-MM-DD_<模块>_<实验名>.md`，例如 `2026-05-15_首页_新版引导.md`

## 模板

```markdown
# A/B 实验：<实验名>

## 假设
> 我们认为 ... 会让指标 ... 提升 X%。

## 设计
- 分组：A 组 = 控制（现有版本），B 组 = 实验（新版本）
- 流量分配：50/50（或其他）
- 持续时间：N 天 / 直到统计显著
- 主指标：
- 护栏指标：

## 结果
| 指标 | A | B | 差异 | p 值 |

## 结论
- 全量 / 回滚 / 继续观察
- 学到了什么

## 后续行动
- [ ] 下一步实验 / 优化
```

## 工具

GrowingIO / Optimizely / 自建分流网关；本目录只存结论与决策快照。
