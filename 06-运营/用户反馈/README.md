# 用户反馈

收集并归档来自客服工单、应用商店评论、社群、用户访谈、NPS 调研的反馈。

## 文件组织建议

- `YYYY-MM_客服工单摘要.md`：每月一份
- `YYYY-MM_用户访谈.md`：每次访谈一份
- `YYYY-MM_NPS-调研结果.md`：NPS / 满意度调研

## 反馈分类标签

| 标签 | 含义 |
|---|---|
| bug | 功能不符合预期 |
| feature-req | 新功能诉求 |
| ux | 体验 / 交互问题 |
| performance | 性能 / 速度抱怨 |
| pricing | 定价 / 付费相关 |
| onboarding | 新手引导问题 |

## 处理流程

1. 收集 → 本目录归档（脱敏：用户名/手机号打码）
2. 分类 → 打标签
3. 优先级排序（影响人数 × 严重度）
4. 高优反馈 → 立 issue → 进 [03-开发/Sprint](../../03-开发/Sprint%20计划与回顾/)
5. 处理完 → 闭环回复用户
