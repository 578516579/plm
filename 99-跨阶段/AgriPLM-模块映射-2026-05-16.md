# AgriPLM·AI → PLM 领域映射 v1

> **来源**: AgriPLM-AI-PRD-完整版.docx (v3.0, 26545 字, 30 模块) + 31 个 HTML 原型
> **目标**: 把 AgriPLM·AI(农业 PM SaaS)\` 30 个模块映射到通用 PLM(internal-tool / solo / early)
> **决策**: 农业领域内容(AgriKB / IoT / 土壤 / 灌溉 / 病虫害 / 气象站) **全部剥离**,落 v 后续 backlog;通用 PM 工作流模块进 MVP

---

## 0. 映射规则

| AgriPLM 元素 | PLM 处理 |
|---|---|
| 农业知识库 (AgriKB) | 改为**通用知识库**(可选 v0.5+ 启用),MVP 不做 |
| Dify 工作流 / AI 编排 | 全部标 N/A(本期不引入 AI 编排引擎,Phase B 自进化已提到) |
| MCP 集成 (GitLab/飞书/Figma/Jira/钉钉) | 标 v0.5+,MVP 可手动同步 |
| 土壤/气象/IoT/灌溉/病虫害 | **全部剥离**,本 PLM 不涉及农业 |
| 角色: 张总/王工/李工/陈工/赵工 | 本期 solo,统一 = Wjl;角色枚举留作 v0.3+ 扩展 |
| 业务线: 精准农业/农资流通/植保服务/质量溯源 | 不要,用 `domain` 字段空着或自填 |

## 2. PLM v0.x MVP 实体清单 (压缩后)

经过映射,**通用 PLM MVP 只需要 ~6 张业务表**:

| 表名 | 中文 | 状态 | 备注 |
|---|---|---|---|
| `tb_project` | 项目 | ✅ done (v0.1.0) | 5 索引 + 8 字典数据 |
| `tb_requirement` | 需求 | 🔜 v0.2 | **Phase 01 立项中** |
| `tb_task` | 任务 | 🔜 v0.2 | **Phase 01 立项中**;关联 requirement_id + sprint_id |
| `tb_sprint` | 迭代 | 🔜 v0.2 | **Phase 01 立项中**;关联 project_id |
| `tb_defect` | 缺陷 | v0.3 | 关联 sprint_id + task_id |
| `tb_test_case` | 测试用例 | v0.3 | 关联 requirement_id |
| `tb_document` | 通用文档 | v0.4 | `doc_type` 字段统一管 PRD / 设计 / 测试报告 / 手册 |
| `tb_release` | 发布记录 | v0.4 | 关联 project_id + sprint_id |
| `tb_proposal` | 立项建议书 | v0.5 | 当前用 markdown 文件管,转表是可选优化 |

> AgriPLM 30 模块对应 30 张表 + 大量农业字段 → PLM 压缩到 9 张表(去 70%),且 4 张可推迟。

## 3. 显式剥离清单 (10 项,绝对不做)

1. **AgriKB 知识库** (2847 篇农业文档 + Milvus 向量) — 通用 PLM 不需要
2. **Dify 工作流引擎 / 18 个工作流** — 自进化已经替代了 AI 编排
3. **5 个 AI Agent (代码审查/测试生成/...)** — 同上
4. **5 个 MCP Server 集成** — 推到 v0.5+,且评估必要性
5. **土壤传感器 / 气象站 / 灌溉控制器 IoT 协议** — 业务无关
6. **病虫害库 / 作物品种库** — 业务无关
7. **15 维竞品对比矩阵** — internal-tool 无竞品
8. **农业 UI 组件库** (土壤墒情/IoT 设备状态) — 业务无关
9. **OpenAPI / AsyncAPI / GraphQL Spec 生成** — Springdoc 已生成,无需独立模块
10. **Feature Flag 平台** — 用 Unleash/GrowthBook 替代,不重造

## 4. AgriPLM 字段命名 → PLM 字段命名转换

| AgriPLM 原字段 | PLM 字段 | 备注 |
|---|---|---|
| `field_code` (地块编号) | (无) | 不要 |
| `moisture_pct` (土壤含水率) | (无) | 不要 |
| `recommend_time` (推荐灌溉时间) | (无) | 不要 |
| `confidence` (AI 推荐置信度) | (无) | 不要,通用业务表不引入 AI 字段 |
| `business_line` (业务线: 精准农业/农资流通...) | `domain` (空字符串或自填) | 通用化 |
| `owner` (负责人,枚举: 王工/李工/...) | `assignee_user_id` (FK→sys_user) | 接 RuoYi 用户体系 |
| `status` (状态枚举因模块不同) | 各表自己的 `status` 字典 | 沿用 Project 字典模式 |
