## A. 准入条件（进入 Phase 01 前必须满足）

- [ ] 已有明确的业务痛点 / 机会描述（一句话能讲清）
- [ ] 已有初步价值假设（用户量、收入、成本节约等量级估计）
- [ ] 已识别该项目的最终决策人（拍板"是否做"的人）
- [ ] 列入团队 [OKR](../团队%20OKR.md) 候选清单 / 或已与 PM Owner 达成共识立项

---

## B. 必产出物（按顺序产出）

### B.1 市场调研

- [ ] 文件：`01-立项/<模块>-市场调研.md`（基于 [模板](../../01-立项/市场调研.md)）
- [ ] 含：目标市场规模（TAM / SAM / SOM）
- [ ] 含：至少 3 个竞品的功能矩阵
- [ ] 含：至少 5 个用户访谈纪要 / 替代证据
- [ ] 含：明确结论（值得做 / 边做边看 / 不值得做）

### B.2 PRD（产品需求文档）

- [ ] 文件：`01-立项/<模块>-PRD.md`（基于 [模板](../../01-立项/PRD.md)）
- [ ] 含：背景与目标
- [ ] 含：核心衡量指标（北极星指标 / OKR KR 对应）
- [ ] 含：用户角色与典型场景
- [ ] 含：功能需求清单（带优先级 P0/P1/P2）
- [ ] 含：每条需求的验收标准
- [ ] 含：非功能需求（性能 / 安全 / 合规约束）
- [ ] 含：**Out of Scope 明确**（什么本期不做）
- [ ] 含：依赖与约束（上下游系统、第三方）
- [ ] PRD 版本号已分配（v1.0）

### B.3 商业计划（L1 强制 / L2 可省略）

- [ ] 文件：`01-立项/<模块>-商业计划.md`
- [ ] 含：价值主张
- [ ] 含：商业模式 + 收入预测
- [ ] 含：投入预算（人 / 基础设施 / 推广）
- [ ] 含：关键假设 + 验证方法（≥ 3 条）

### B.4 立项评审纪要

- [ ] 文件：`01-立项/<模块>-立项评审纪要-<YYYY-MM-DD>.md`
- [ ] 评审日期：YYYY-MM-DD
- [ ] 评审材料已提前 ≥ 2 个工作日发出
- [ ] 评审结论明确（**通过 / 有条件通过 / 不通过**）

---

## C. Definition of Done（出口 DoD）

- [ ] PRD v1.0 已评审通过（结论"通过"或"有条件通过 + 条件已闭环"）
- [ ] 评审纪要已归档到 `01-立项/`
- [ ] 评审中识别的风险已 100% 录入 [99-跨阶段/风险登记册.md](../风险登记册.md)
- [ ] 已排上当前或下一季度 OKR（[99-跨阶段/团队 OKR.md](../团队%20OKR.md)）
- [ ] 已通知技术 lead（启动 Phase 02 设计的准入条件）
- [ ] **本 Checklist 文件已 commit 入库**（commit message：`docs(gate): <module> phase 01 passed`）

## L2 简化指引

L2 中型需求可按下列方式简化：
- **B.1 市场调研** → 可以是一段 200 字的"竞品速览 + 我们的差异化"，不强制 5 个访谈
- **B.3 商业计划** → 可省略；改为在 PRD 内加一段"投入产出预估"
- **D 评审签字** → 业务 + 产品 + 技术 lead 三方即可（不强求 CFO / 设计）

L3 小型需求**不走本 Gate**，直接走 Phase 03 PR 流程。
