### B.1 系统架构(inline,L2 简化但保留关键内容)

> ⚠️ inline 紧凑型,无独立 02-设计/ 文档,见 E 段豁免。

**C4 Container**

```
┌──────────────────────────────────────────────────────────────┐
│ plm-frontend (Vue 3.5 SPA, port :80)                         │
│   src/views/business/arch/index.vue                          │
│      ↓ axios via @/utils/request                             │
└──────────────────────────────────────────────────────────────┘
            │ POST/GET/PUT /dev-api/business/arch/*
            │ + JWT Bearer
            ▼
┌──────────────────────────────────────────────────────────────┐
│ plm-backend (Spring Boot 4, port :8081)                      │
│   plm-admin/.../ArchController                               │
│      → IArchService (CRUD + aiGenerate)                      │
│         → ArchMapper (MyBatis XML)                           │
│         → ProjectMapper (校验 projectId 存在性)              │
│         → AiService (plm-common,本期路由到 MockAiProvider)   │
└──────────────────────────────────────────────────────────────┘
            │ jdbc:mysql
            ▼
┌──────────────────────────────────────────────────────────────┐
│ MySQL 8.x · schema plm                                       │
│   tb_arch (架构方案主表)                                     │
│   sys_dict_data (7 字典: biz_arch_mode/stack/database/       │
│                  ai_engine/deployment/iot_protocol/status)   │
│   sys_menu (业务菜单挂载点 menu_id 2230-2235 @ 需求与设计阶段)│
└──────────────────────────────────────────────────────────────┘
```

**组件清单**

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-arch` (Maven) | 后端模块 | Spring Boot 4 + MyBatis | Wjl |
| `views/business/arch/index.vue` | 前端主视图(列表 + 编辑 modal + AI 生成对话框 + Mermaid C4 渲染) | Vue 3.5 + Element Plus 2.13 + mermaid.js | Wjl |
| `tb_arch` (MySQL) | 架构方案主数据 | MySQL 8 utf8mb4 | Wjl |
| `biz_arch_*` (7 字典) | 架构模式 / 技术栈 / 数据库 / AI 编排 / 部署 / IoT / 状态 | sys_dict_data | Wjl |
| `aiGenerate` 方法 | AI HLD 生成(mock + AiService 审计):designContent + c4DiagramContent + nfrMapping 三段 | AiService 门面 | Wjl |

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/arch
       → JwtAuthenticationTokenFilter
       → ArchController.add (PreAuthorize business:arch:add)
       → ArchServiceImpl.insertArch
       → 字段校验 (title/projectId/authorUserId) + projectMapper.selectProjectById 存在性检查 (702)
       → 6 ENUM 白名单 (mode/stack/database/ai_engine/deployment/iot_protocol → 604)
       → 默认值 (aiGenerated=N, status='00') + 新建 status 非草稿拒 (601)
       → generateArchNo (ARCH-YYYY-NNNN)
       → mapper.insert (DuplicateKey 重试 1 次)
       → 返回 arch_id

AI 生成 → Vue 触发 → POST /business/arch/ai/generate/{id}
       → ArchServiceImpl.aiGenerate
       → aiService.chat(AiChatRequest) [审计落表]
       → mock 输出 designContent (Markdown HLD) + c4DiagramContent (Mermaid C4 Container) + nfrMapping (4 维 NFR)
       → mapper.update(designContent/c4DiagramContent/nfrMapping/aiGenerated/aiGeneratedAt)
       → 返回 Arch

状态推进 → Vue 切 status → PUT /business/arch
       → ArchServiceImpl.updateArch → STATUS_TRANSITIONS 校验 (4 态含反向边)
         00 → 01
         01 → {00 反向打回 / 02 已确认}
         02 → 03
         03 (终态)
       → 非法转换 → 601
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务。

**集成点**:sys_user(author/reviewer FK)/ sys_dict_data(7 字典) / sys_menu(2230-5)/ sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log** / **plm-project 项目存在性校验** / **plm-prd prdId 可空引用**。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 archdesign.html):

```
旅程 1 - 新建架构方案 + AI HLD(主流量)
  Step1 用户进入 /business/arch
  Step2 点新增 → 弹 modal 填表(标题 + 关联项目 + 6 维技术选型 ENUM 下拉)
  Step3 保存 → 后端 INSERT → 编号 ARCH-YYYY-NNNN → status='00' 草稿
  Step4 列表 row 点"AI 生成" → 后端 aiGenerate
  Step5 modal 展示 designContent (HLD Markdown) + c4DiagramContent (Mermaid C4 渲染) + nfrMapping (4 维 NFR 表格)

旅程 2 - 评审打回(反向边 01→00 特色)
  Step1 PUT { status: '01' } → 提交评审
  Step2 评审者发现技术栈选错 → PUT { status: '00' } 打回草稿
  Step3 ⚠ 这是 arch / dbdesign / apidesign / prd / ued 5 模块的共用 4 态含反向边特性

旅程 3 - 架构方案 lifecycle
  Step1 PUT { status: '02' } → 评审中 → 已确认(冻结,作为 dbdesign/apidesign 上游)
  Step2 PUT { status: '03' } → 已确认 → 已废弃(终态,新版本另建方案)
  Step3 ⚠ 02 → 00 / 02 → 01 反向非法(确认后不能回退到评审/草稿,只能废弃后另建)→ 单测覆盖

旅程 4 - 下游引用追踪(P2 v0.2)
  Step1 dbdesign / apidesign 创建时引用 archId
  Step2 已确认架构方案的引用计数(下迭代)
  Step3 ⏳ v0.2 实现
```
