### B.1 系统架构(inline,L2 简化但保留关键内容)

> ⚠️ inline 紧凑型,无独立 02-设计/ 文档,见 E 段豁免。

**C4 Container**

```
┌──────────────────────────────────────────────────────────────┐
│ plm-frontend (Vue 3.5 SPA, port :80)                         │
│   src/views/business/competitive/index.vue                   │
│      ↓ axios via @/utils/request                             │
└──────────────────────────────────────────────────────────────┘
            │ POST/GET/PUT /dev-api/business/competitive/*
            │ + JWT Bearer
            ▼
┌──────────────────────────────────────────────────────────────┐
│ plm-backend (Spring Boot 4, port :8081)                      │
│   plm-admin/.../CompetitiveController                        │
│      → ICompetitiveService (CRUD + aiAnalyze)                │
│         → CompetitiveMapper (MyBatis XML)                    │
│         → ProjectMapper (校验 projectId 存在性)              │
│         → AiService (plm-common,本期路由到 MockAiProvider)   │
└──────────────────────────────────────────────────────────────┘
            │ jdbc:mysql
            ▼
┌──────────────────────────────────────────────────────────────┐
│ MySQL 8.x · schema plm                                       │
│   tb_competitive (竞品主表)                                  │
│   sys_dict_data (2 字典: biz_competitive_tier/status)        │
│   sys_menu (业务菜单挂载点 menu_id 2220-2225 @ 规划阶段)     │
└──────────────────────────────────────────────────────────────┘
```

**组件清单**

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-competitive` (Maven) | 后端模块 | Spring Boot 4 + MyBatis | Wjl |
| `views/business/competitive/index.vue` | 前端主视图(列表 + 编辑 modal + AI 分析对话框) | Vue 3.5 + Element Plus 2.13 | Wjl |
| `tb_competitive` (MySQL) | 竞品主数据 | MySQL 8 utf8mb4 | Wjl |
| `biz_competitive_*` (2 字典) | 价格档 / 状态枚举 | sys_dict_data | Wjl |
| `aiAnalyze` 方法 | AI SWOT + 综合报告生成(mock + AiService 审计) | AiService 门面 | Wjl |

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/competitive
       → JwtAuthenticationTokenFilter
       → CompetitiveController.add (PreAuthorize business:competitive:add)
       → CompetitiveServiceImpl.insertCompetitive
       → 字段校验 (competitorName/projectId/authorUserId) + projectMapper.selectProjectById 存在性检查
       → ENUM 白名单 (pricingTier ∈ {free,midrange,enterprise})
       → 默认值 + generateCompetitiveNo
       → mapper.insert (DuplicateKey 重试 1 次)
       → 返回 competitive_id

AI 分析 → Vue 触发 → POST /business/competitive/ai/analyze/{id}
       → CompetitiveServiceImpl.aiAnalyze
       → aiService.chat(AiChatRequest) [审计落表]
       → mock 输出 SWOT 四象限 + Markdown 综合报告
       → mapper.update(strengths/weaknesses/opportunities/threats/aiAnalysisReport/aiGenerated/aiGeneratedAt)
       → 返回 Competitive
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务。

**集成点**:sys_user(author FK)/ sys_dict_data / sys_menu / sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log** / **plm-project 项目存在性校验**。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 competitive.html):

```
旅程 1 - 新建竞品 + AI SWOT(主流量)
  Step1 用户进入 /business/competitive
  Step2 点新增 → 弹 modal 填表(竞品名 + 厂商 + 网址 + 价格档 + 项目 + 功能矩阵)
  Step3 保存 → 后端 INSERT → 编号生成 → status='00' 草稿
  Step4 列表 row 点"AI 分析" → 后端 aiAnalyze
  Step5 modal 展示 SWOT 四象限 + Markdown 综合报告

旅程 2 - 历史竞品审阅
  Step1 列表筛选(项目 / 状态 / 价格档)
  Step2 点"详情" → modal 展示完整字段
  Step3 编辑后保存 → status 保持 / 显式切换

旅程 3 - 竞品 lifecycle
  Step1 PUT { status: '01' } → 草稿 → 已发布(发布上架)
  Step2 PUT { status: '02' } → 已发布 → 已归档(终态)
  Step3 ⚠ 无反向边:发布后不可回草稿,归档后不可重启 → 单测覆盖

旅程 4 - 订阅监控(P2,字段已预留,本期 UI 不实现)
  Step1 PUT { monitor_enabled: 'Y', monitor_keywords: 'agri,Jira clone' }
  Step2 ⏳ Phase 04+ 定时爬虫接入
```
