## B. 必产出物

### B.1 系统架构（inline，依 L2 简化指引但保留 L1 关键内容）

> ⚠️ 本模块按 L1 但 inline 紧凑型架构（无独立 02-设计/ 文档），原因见 E 段（豁免说明 1）。

**C4 Container（dashboard 模块在 PLM 生态中的位置）**

```
┌──────────────────────────────────────────────────────────────┐
│ plm-frontend (Vue 3.5 SPA, port :80)                         │
│   packages/plm-dashboard/views/index.vue                     │
│      ↓ axios via @/utils/request                             │
└──────────────────────────────────────────────────────────────┘
            │ GET /dev-api/business/dashboard/aggregate
            │ + JWT Bearer
            ▼
┌──────────────────────────────────────────────────────────────┐
│ plm-backend (Spring Boot 4, port :8081)                      │
│   plm-admin/.../DashboardController                          │
│      → IDashboardService (aggregate / CRUD)                  │
│         → DashboardMapper (MyBatis XML)                      │
│         → [Future] ProjectMapper / TaskMapper / DefectMapper │
└──────────────────────────────────────────────────────────────┘
            │ jdbc:mysql
            ▼
┌──────────────────────────────────────────────────────────────┐
│ MySQL 8.x · schema plm                                       │
│   tb_dashboard (用户工作台预设)                                │
│   sys_dict_data (biz_dashboard_status, biz_dashboard_widget) │
│   sys_menu (业务菜单挂载点)                                    │
└──────────────────────────────────────────────────────────────┘
```

**组件清单**：

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-dashboard` (Maven) | 后端模块：domain/mapper/service/Controller | Spring Boot 4 + MyBatis | Wjl |
| `@plm/dashboard` (npm pkg) | 前端模块：index.vue + router | Vue 3.5 + Element Plus 2.13 + ECharts | Wjl |
| `tb_dashboard` (MySQL 表) | 用户自定义 widget 布局存储 | MySQL 8 utf8mb4 | Wjl |
| `biz_dashboard_*` (字典) | widget 类型 / 状态枚举 | sys_dict_data | Wjl |
| `aggregate()` Service 方法 | 6 类 widget 聚合查询（本期 mock，下期接真实跨模块） | Java 17 + Stream | Wjl |

**关键业务路径（端到端时序）**

```
用户 → 浏览器 → Vue Dashboard 视图 onMounted
                  ↓
              api.aggregate(ownerUserId)
                  ↓
              GET /dev-api/business/dashboard/aggregate
                  ↓ JwtAuthenticationTokenFilter 鉴权
              DashboardController.aggregate
                  ↓ @PreAuthorize('business:dashboard:query')
              DashboardServiceImpl.aggregate
                  ↓ mock 6 widget
              返回 {stats, activeProjects, myTodos, qualitySnapshot, aiMetrics, lifecycle}
                  ↓
              Vue 渲染 6 类卡片
              60 秒刷新（refreshInterval default）
```

**部署拓扑**：dev/staging/prod 均为单实例（无横向扩展需求，aggregate 接口 mock 阶段无压力）；future 真聚合阶段需加 Redis 缓存（详见 ADR 占位待写）。

**与现有 PLM 集成点**：sys_user（owner_user_id FK）/ sys_dict_data（widget 类型字典）/ sys_menu（菜单挂载，本期 SQL 待补）/ sys_oper_log（CRUD 操作审计）/ JWT 鉴权链。

### B.4 UI 设计

- [x] 用户旅程图（inline，按原型 dashboard.html 转 Vue 的映射）：

```
旅程 1 - 首次登录看首屏（80% 流量）
  Step1 用户登录 → 跳 /business/dashboard
  Step2 view onMounted → loading
  Step3 axios GET aggregate
  Step4 渲染 6 卡片（统计+在办+待办+生命周期+质量+AI）
  Step5 用户停留浏览 / 点统计跳详情 / 点待办跳任务

旅程 2 - 跨模块快速跳转（次高频）
  Step1 在工作台主屏看到「在办项目 7」
  Step2 点击数字 → 跳 /business/project
  Step3 项目列表带筛选 status=active

旅程 3 - 周期性刷新（被动）
  Step1 60 秒计时器触发
  Step2 重新 aggregate
  Step3 更新统计卡 + 待办（保留其他静态）

旅程 4 - AI 命令栏占位（本期仅 UI）
  Step1 用户点击 AI 输入栏
  Step2 弹框提示「ai-agent 模块就绪后启用」
  Step3 用户跳其他模块
```
