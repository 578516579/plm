**组件清单**

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-dbdesign` (Maven) | 后端模块 | Spring Boot 4 + MyBatis | Wjl |
| `packages/plm-dbdesign/src/views/index.vue` | 前端主视图(列表 + 编辑 modal + AI 生成对话框) | Vue 3.5 + Element Plus 2.13 | Wjl |
| `tb_dbdesign` (MySQL) | DB 设计主数据(ER / 字典 / DDL / 规范检查 LONGTEXT) | MySQL 8 utf8mb4 | Wjl |
| `biz_dbdesign_*` (2 字典) | DB 引擎 / 状态枚举 | sys_dict_data | Wjl |
| `aiGenerate` 方法 | AI 输出 Mermaid ER + Markdown 字典 + CREATE TABLE + 规范 JSON(mock + AiService 审计) | AiService 门面 | Wjl |

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/dbdesign
       → JwtAuthenticationTokenFilter
       → DbDesignController.add (PreAuthorize business:dbdesign:add)
       → DbDesignServiceImpl.insertDbDesign
       → 字段校验 (title/projectId/authorUserId) + projectMapper.selectProjectById 存在性检查
       → ENUM 白名单 (dbEngine ∈ {mysql,postgresql,kingbase})
       → 默认值 + generateDbDesignNo
       → mapper.insert (DuplicateKey 重试 1 次)
       → 返回 dbdesign_id

AI 生成 → Vue 触发 → POST /business/dbdesign/ai/generate/{id}
       → DbDesignServiceImpl.aiGenerate
       → aiService.chat(AiChatRequest) [审计落表]
       → mock 输出 Mermaid ER + Markdown 字典 + CREATE TABLE + 规范 JSON
       → mapper.update(erDiagramContent/dataDictionary/ddlScript/normalizationCheck/aiGenerated/aiGeneratedAt)
       → 返回 DbDesign
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务。

**集成点**:sys_user(author + reviewer 双 FK)/ sys_dict_data / sys_menu / sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log** / **plm-project 项目存在性校验**;arch_id 字段已预留但 plm-arch 仍是空壳,本期不做 archId 存在性校验。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 dbdesign.html):

```
旅程 1 - DBA 一键生成 ER + DDL(主流量)
  Step1 用户进入 /business/dbdesign
  Step2 点新增 → 弹 modal 填表(设计标题 + 项目 + DB 引擎 + DBA)
  Step3 保存 → 后端 INSERT → 编号 DB-YYYY-NNNN → status='00' 草稿
  Step4 列表 row 点"AI 生成" → 后端 aiGenerate
  Step5 modal 三联展示:ER 图(Mermaid 渲染)+ 数据字典(Markdown 表格)+ DDL 代码块 + 规范检查 JSON

旅程 2 - DB 设计评审 + 反向打回(关键反向边场景)
  Step1 DBA 提交评审 PUT { status: '01' } → 草稿 → 评审中
  Step2 评审者发现规范有问题 → PUT { status: '00' } 反向打回草稿
  Step3 DBA 修改后再次提交 PUT { status: '01' }
  Step4 通过后 PUT { status: '02' } → 评审中 → 已确认
  Step5 上线后废弃 PUT { status: '03' } → 已废弃(终态)

旅程 3 - 后端开发者消费 DDL
  Step1 列表筛选(项目 / 状态='02 已确认')
  Step2 点"详情" → modal 展示完整字段
  Step3 复制 DDL 到 plm-backend/sql/business-<entity>.sql

旅程 4 - 多次 AI 迭代(P2,本期支持单次覆盖,版本字段留 Phase 04+ 加)
  Step1 重新 POST /ai/generate/{id} → 覆盖上一版 ER / 字典 / DDL
```
