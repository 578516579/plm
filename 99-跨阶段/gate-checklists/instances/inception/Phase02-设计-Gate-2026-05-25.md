## B. 必产出物

### B.1 系统架构(inline,L2 简化但保留关键内容)

> ⚠️ inline 紧凑型,无独立 02-设计/ 文档,见 E 段豁免。

**C4 Container**

```
┌──────────────────────────────────────────────────────────────┐
│ plm-frontend (Vue 3.5 SPA, port :80)                         │
│   src/views/business/inception/index.vue                     │
│      ↓ axios via @/utils/request                             │
└──────────────────────────────────────────────────────────────┘
            │ POST/GET/PUT /dev-api/business/inception/*
            │ + JWT Bearer
            ▼
┌──────────────────────────────────────────────────────────────┐
│ plm-backend (Spring Boot 4, port :8081)                      │
│   plm-admin/.../InceptionController                          │
│      → IInceptionService (CRUD + aiGenerate)                 │
│         → InceptionMapper (MyBatis XML)                      │
│         → AiService (plm-common,本期路由到 MockAiProvider)   │
└──────────────────────────────────────────────────────────────┘
            │ jdbc:mysql
            ▼
┌──────────────────────────────────────────────────────────────┐
│ MySQL 8.x · schema plm                                       │
│   tb_inception (立项主表)                                    │
│   sys_dict_data (3 字典: biz_line/type/status)               │
│   sys_menu (业务菜单挂载点 menu_id 2316-2321)                │
└──────────────────────────────────────────────────────────────┘
```

**组件清单**

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-inception` (Maven) | 后端模块 | Spring Boot 4 + MyBatis | Wjl |
| `views/business/inception/index.vue` | 前端主视图(双卡片 + 历史表) | Vue 3.5 + Element Plus 2.13 | Wjl |
| `tb_inception` (MySQL) | 立项主数据 | MySQL 8 utf8mb4 | Wjl |
| `biz_inception_*` (3 字典) | 业务线 / 类型 / 状态枚举 | sys_dict_data | Wjl |
| `aiGenerate` 方法 | AI 立项建议书生成(mock + AiService 审计) | AiService 门面 | Wjl |

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/inception
       → JwtAuthenticationTokenFilter
       → InceptionController.add (PreAuthorize business:inception:add)
       → InceptionServiceImpl.insertInception
       → 字段校验 + 字典白名单 + 默认值 + generateInceptionNo
       → mapper.insert (DuplicateKey 重试 1 次)
       → 返回 inception_id

AI 生成 → Vue 触发 → POST /business/inception/ai/generate/{id}
       → InceptionServiceImpl.aiGenerate
       → aiService.chat(AiChatRequest) [审计落表]
       → mock 输出 Markdown proposal + risks
       → mapper.update(aiGenerated/proposal/risks/aiGeneratedAt)
       → 返回 Inception
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务。

**集成点**:sys_user(submitter/approver FK)/ sys_dict_data / sys_menu / sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log**。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 inception.html):

```
旅程 1 - 新建立项 + AI 一键生成(主流量)
  Step1 用户进入 /business/inception
  Step2 左卡填表(项目名称 + 业务线 + 类型 + 背景 + 工期 + 团队)
  Step3 点"保存并 AI 分析"
  Step4 后端 INSERT → 编号生成 → AiService 审计 → mock Markdown
  Step5 右卡渲染 Markdown 建议书 + 风险红警条

旅程 2 - 历史立项审阅
  Step1 滚动到底部历史表
  Step2 点"载入编辑" → 表单回填
  Step3 编辑后保存 → status='00' 保持草稿

旅程 3 - AI 二次分析
  Step1 历史表 row 点"AI 分析"按钮
  Step2 后端覆盖更新 aiProposalContent / aiRisks / aiGeneratedAt
  Step3 列表刷新

旅程 4 - 立项审批(预留接口,本期 UI 简化)
  Step1 后端 PUT { status: '01' } → 提交
  Step2 后端 PUT { status: '02' } → 审批中
  Step3 后端 PUT { status: '03' } → 已批准(自动 approvedAt)
        或 PUT { status: '04', rejectReason: '...' } → 已驳回
  Step4 反向边 PUT { status: '00' } 重新打回(仅 04→00 允许)
```
