#### D.3.2 NPM 依赖（npm audit `npm audit --omit=dev --registry=https://registry.npmjs.org`）

| CVE 类 | 包 | 严重度 | 数量 | 处置 |
|---|---|---|---|---|
| Axios CRLF / SSRF / Prototype Pollution / DoS 等 | `axios <1.16.1` | **high** | 13 | 🟡 已知 risk |
| Glob CLI command injection | `glob 10.2.0-10.4.5` | **high** | 1 | 🟡 已知 risk |

**实际暴露面评估**（按 PLM `internal-tool` + 后端 URL 不接受用户输入透传）:

- ✅ Axios CRLF/SSRF：PLM 前端不允许用户输入任意 URL,所有 axios 调用都通过 `@/utils/request` 走固定后端 baseURL,**实际暴露面 0**
- ✅ Glob CLI：仅 dev/build 依赖（vite/playwright 工具链）,生产 jar 不含,**实际暴露面 0**

→ **有条件通过**，技术债登记到下一 Sprint backlog：
- `chore(deps): bump axios from 1.x to 1.16.1+ (CVE-2025-62718 etc.)`
- `chore(deps): bump glob from 10.x to 11.x (CVE GHSA-5j98-mcp5-4vw2)`

#### D.3.3 OWASP Top 10 抽查

- [x] **A01 Broken Access Control**：每端点 `@PreAuthorize`,@ss.hasPermi 函数校验
- [x] **A02 Cryptographic Failures**：JWT secret 走环境变量 `${JWT_SECRET}`,无默认值
- [x] **A03 Injection**：MyBatis 全部走 `#{param}`,无 `${param}` 字符串拼接
- [x] **A05 Security Misconfiguration**：CORS/CSRF 通过 Spring Security 标准配置
- [x] **A07 Identification and Authentication Failures**：JWT + captcha 双重防护
- [x] **A09 Security Logging and Monitoring**：`@Log` 注解 + Druid SQL 日志
- [ ] **A04 Insecure Design** / **A08 Software Integrity** / **A10 SSRF**：留 Phase 05 前再走
