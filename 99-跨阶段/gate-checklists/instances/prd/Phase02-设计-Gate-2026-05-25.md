**组件清单**

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/prd
       → JwtAuthenticationTokenFilter
       → PrdController.add (PreAuthorize business:prd:add)
       → PrdServiceImpl.insertPrd
       → 字段校验 + ENUM 白名单(scene/target_user) + projectId 702 校验
       → 默认值(version=v1.0 / aiGenerated=N / status=00) + generatePrdNo(PRD-YYYY-NNNN)
       → mapper.insert (DuplicateKey 重试 1 次)
       → 返回 prd_id

AI 生成 → Vue 触发 → POST /business/prd/ai/generate/{id}
       → PrdServiceImpl.aiGenerate
       → aiService.chat(AiChatRequest with 场景化 prompt) [审计落表 ai_invocation_log]
       → 12 组合场景化 mock 输出 7 段 Markdown
       → computeCompleteness(content) → 真实分数(7 段命中 × 90% + 字数 × 10%)
       → AiChatResult.failure? → WARN + fallback 基础模板(业务不阻塞)
       → mapper.update(aiGenerated=Y/content/completenessScore/aiGeneratedAt)
       → 返回 Prd
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务。

**集成点**:sys_user(author/reviewer FK)/ sys_dict_data / sys_menu / sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log** / tb_project(FK 702)。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 prd.html):

```
旅程 1 - 新建 PRD + AI 一键生成(主流量)
  Step1 用户进入 /business/prd
  Step2 左卡填表(功能名称 + 业务场景 + 目标用户 + 自然语言描述)
  Step3 点"AI 生成完整 PRD"按钮
  Step4 后端 INSERT(如无 ID)或 UPDATE → 编号生成 → AiService 审计调用 → 场景化 mock 7 段 Markdown
  Step5 右卡渲染 Markdown 预览 + 完整度评分条(目标 ≥ 80%)

旅程 2 - 历史 PRD 评审与状态推进
  Step1 列表选 PRD → 详情查看 7 段 + 完整度
  Step2 提交评审 → PUT { status: '01' }
  Step3 审核人评审 → PUT { status: '02' }(已确认)
        或 PUT { status: '00' }(反向打回到草稿)

旅程 3 - PRD 二次 AI 重生成
  Step1 历史列表点"AI 重生成"
  Step2 后端覆盖更新 content / completenessScore / aiGeneratedAt
  Step3 列表/详情刷新

旅程 4 - PRD 废弃(终态)
  Step1 在 02 已确认状态点"废弃"
  Step2 PUT { status: '03' } → 已废弃(终态,不可恢复)
```
