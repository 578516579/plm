### Phase 03 E 段豁免反弹的诊断

Phase 03 模板对 solo 模式的 friction 暴露出来：
- **F-P03-01**: B.4 测试代码要求"Service 覆盖率 ≥ 70%"对 solo 单 Sprint 不现实，事实上推迟到 Phase 04 才合理。模板应区分"代码合入"和"代码 + 测试合入"两种 DoD
- **F-P03-02**: C Sprint 计划/回顾文件在 solo 模式下与 Gate 实例重复，应允许合并到 Gate 实例
- **F-P03-03**: B.3 "SQL 在 staging 演练成功"对没 staging 的项目早期不适用

这 3 个 friction 是 0001/0002/0003 未覆盖的新维度（**软件成熟度 / 测试时机** 维度）。

→ **触发新 proposal 候选**（写到 Phase 03 reflect 时正式提出）：
- 0004：B.4 测试代码"代码骨架/含测试" 两阶段 DoD
- 0005：solo 模式 Sprint 文档与 Gate 实例可合并
