## B. ER / DDL 设计

```
tb_requirement_review:
  review_id           BIGINT(20)  PK AUTO_INCREMENT
  requirement_id      BIGINT(20)  NOT NULL  FK→tb_requirement.requirement_id (idx)
  reviewer_user_id    BIGINT(20)  NOT NULL  FK→sys_user.user_id (idx)
  review_result       VARCHAR(2)  NOT NULL  dict biz_req_review_result (idx)
  review_comment      VARCHAR(1000)         打回必填
  review_at           DATETIME              提交时回填
  + RuoYi 标准 6 字段

tb_prd ALTER:
  ADD COLUMN requirement_id BIGINT(20) DEFAULT NULL AFTER project_id
  ADD INDEX idx_prd_requirement (requirement_id)
```

## C. API 设计

| 方法 | 端点 | 权限 | 说明 |
|---|---|---|---|
| GET | `/business/requirement/{id}/reviews` | `business:requirement:query` | 评审历史(倒序) |
| GET | `/business/requirement/review/{reviewId}` | `business:requirement:query` | 单条 |
| POST | `/business/requirement/{id}/review` | `business:requirement:review` | 提交 |
| DELETE | `/business/requirement/review/{reviewIds}` | `business:requirement:review` | 撤回(逻辑删除) |

校验规则:
- reviewResult 白名单 {00, 01},否则 604
- 打回(01)reviewComment 必填(空白也算空),否则 604
- 需求不存在 → 704
- reviewerUserId 未填兜底为当前登录用户;无登录则 602
