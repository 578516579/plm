## D. 风险与技术债

| # | 类型 | 描述 | 处置 |
|---|---|---|---|
| 1 | 风险 (P1) | 看板视图聚合查询性能,1k 任务时是否仍 < 800ms 未实测 | Phase 03 必须 EXPLAIN + 用 1k mock 数据压测 |
| 2 | 风险 (P2) | 6×6 状态机有 36 case,单测覆盖率必须 100%,容易遗漏边 | Phase 03 单测**强制**每条边一个 case,允许 + 禁止都要 |
| 3 | 风险 (P2) | `02 ↔ 01` 双向边(代码评审 → 开发中)在 UI 上可能让 PM/开发误操作 | Phase 03 前端在状态切换前加 confirm 弹窗 |
| 4 | 技术债 | `mr_url` 是单字段,无法关联多个 MR;未来 task 有多 MR 场景要建关联表 | 接受,留 v0.3 改为 `tb_task_mr` 多对多 |
| 5 | 技术债 | `description` 用 TEXT,列表查询时应排除该列(否则慢) | Phase 03 在 Mapper.xml 显式 SELECT 列清单,不要 `SELECT *` |
| 6 | 风险 (P2) | 看板端点 `kanban` 与 Sprint 详情下"任务列表"功能重叠,容易让前端混乱 | Phase 03 前端 review 时确认调用决策树 |
