### B.3 商业计划 — ROI 模型

- [x] 价值主张 — 设计稿统一沉淀 + Figma 集成预留 + AI 设计规范评审,降低设计/开发对接成本与规范违规风险
- [x] **ROI 工时节约模型**:
  - 投入:1 人 × 1.5 天 ≈ 0.225 万元(Wjl + Claude;前后端 + AI mock + 农业组件库设计)
  - 节约:≈ 30 设计稿/年 × 1 小时/稿(找版本 + 走查规范)× 200 元/小时 ≈ 0.6 万元/年(保守值,Phase 04 接 Figma MCP + Dify 真接入后 ROI 倍增)
  - 回本周期:≈ 16 个月(早期阶段,组件库标签复用率上来后回本周期缩短)
- [x] 关键假设 + 验证:
  - **H1**:设计师接受 AI mock 评审 2 个迭代窗口 → 30 天反馈跟踪
  - **H2**:4 态状态机(含反向打回)覆盖 95% 设计稿 lifecycle 场景 → 上线后跟踪 01→00 反向边触发率
  - **H3**:农业 UI 组件 5 类 + CSV 标签满足 80% 农业项目选型 → 复用率 ≥ 60% 算成功
  - **H4**:UED-YYYY-NNNN 编号能稳定不撞号 → 单测已覆盖 DuplicateKeyException 重试
- [x] 退出条件:若 AI 设计规范评审用户保留率 < 20%,转为纯规范 checklist 模式;若 Figma MCP 服务接入成本 > 收益,P2 永久退化为人工填 URL
