```
┌──────────────────────────────────────────────────────────────┐
```

**组件清单**

| 组件 | 职责 | 技术栈 | Owner |
|---|---|---|---|
| `plm-ued` (Maven) | 后端模块 | Spring Boot 4 + MyBatis | Wjl |
| `views/business/ued/index.vue` | 前端主视图(双列 grid + AI 评审报告 + 农业组件库 + Figma 同步 Dialog) | Vue 3.5 + Element Plus 2.13 | Wjl |
| `tb_ued` (MySQL) | UED 设计稿主数据 | MySQL 8 utf8mb4 | Wjl |
| `biz_ued_status` (1 字典) | 4 状态枚举 | sys_dict_data | Wjl |
| `aiReview` 方法 | AI 设计规范评审 + 可用性建议(mock + AiService 审计) | AiService 门面 | Wjl |

**关键路径时序**

```
用户填表 → Vue formRef.validate() → POST /business/ued
       → JwtAuthenticationTokenFilter
       → UedController.add (PreAuthorize business:ued:add)
       → UedServiceImpl.insertUed
       → 字段校验 (title/projectId/designerUserId) + projectMapper.selectProjectById 存在性检查
       → 默认值 (aiGenerated='N' / status='00') + 新建非 00 拒
       → generateUedNo (UED-YYYY-NNNN, 撞号重试 1 次)
       → mapper.insert
       → 返回 ued_id

AI 评审 → Vue 触发 → POST /business/ued/ai/review/{id}
       → UedServiceImpl.aiReview
       → aiService.chat(AiChatRequest) [审计落表]
       → mock 输出 设计规范报告 Markdown + complianceCheck JSON + usabilityIssues + aiReviewScore=85.00
       → mapper.update(aiReviewReport/complianceCheck/usabilityIssues/aiReviewScore/aiGenerated='Y'/aiGeneratedAt)
       → 返回 Ued

Figma 同步 (前端 mock) → Vue figmaSyncDialog → POST /business/ued (走标准 add)
       → 正则解析 figma.com/file/<key> 提取 fileKey
       → versionLabel='v1.0' 默认 + title=`Figma 同步 ${fileKey}`
       → 真接入 v0.6 改走 MCP server tools/call
```

**部署拓扑**:dev/staging/prod 单实例;AI 走 MockAiProvider 兜底,不阻塞业务;Figma MCP 端点字段预留,无远程依赖。

**集成点**:sys_user(designerUserId/reviewerUserId FK)/ sys_dict_data / sys_menu / sys_oper_log / JWT 链 / **AiService 审计表 ai_invocation_log** / **plm-project 项目存在性校验** / **plm-requirement(可空关联追溯,延迟一致性)**。

### B.4 UI 设计

- [x] 用户旅程(inline,对齐原型 ued.html):

```
旅程 1 - 新建设计稿 + AI 设计规范评审(主流量)
  Step1 用户进入 /business/ued
  Step2 点新增 → 弹 modal 填表(关联项目 + 设计稿名 + Figma URL + fileKey + 版本 + 农业组件标签)
  Step3 保存 → 后端 INSERT → 编号生成 → status='00' 草稿
  Step4 列表 row 点选 → 右侧空状态变为「点击 AI 规范检查」
  Step5 点「✨ AI 设计规范检查」 → 后端 aiReview → 右侧渲染 Markdown 报告 + 可用性 alert + 评审分

旅程 2 - Figma 链路同步(mock,v0.6 真接入)
  Step1 点「📎 从 Figma 同步设计稿」 → 弹 dialog
  Step2 填 Figma URL + 关联项目
  Step3 提交 → 前端正则解析 fileKey 写库,versionLabel='v1.0' 默认
  Step4 提示「mock,真接入留待 v0.6」

旅程 3 - 4 态状态机(含反向打回)
  Step1 PUT { status: '01' } → 草稿 → 评审中
  Step2 PUT { status: '00' } → 评审中 → 草稿(反向打回,UED 特色)
  Step3 PUT { status: '02' } → 评审中 → 已确认
  Step4 PUT { status: '03' } → 已确认 → 已废弃
  Step5 ⚠ 02→01 反向边非法 / 00→02 跳级非法 / 03 终态保护 → 单测覆盖

旅程 4 - 农业 UI 组件库(对齐原型 §3 5 张卡片)
  Step1 进入 /business/ued → 看见底部 5 张组件卡(大屏/移动农事/IoT/地图/作物周期)
  Step2 新建/编辑时点击卡 → 双向勾选 agri_component_tags CSV
  Step3 列表展示已勾选组件 tag(下迭代加筛选)
```
