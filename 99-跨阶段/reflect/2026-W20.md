### 1.3 不顺手的部分 ⚠️ — 找到 5 类新模式

#### F-W20-01: 候选 proposal 堆积，反思 → proposal 通道堵塞

- **现象**: signals/2026-05.md 累积 28 条 `0007 ~ 0034` 候选条目，但 proposals/ 目录止步于 0006。candidates 没有自动升格机制，只在月末"统一回看"——结果是已经过了 Phase 05/06 的 friction，到现在还在候选状态。
- **影响**: 等于反思了但没行动，违反 [reflect/README.md 反模式](README.md#反模式) "反思变成心理按摩"。
- **根因**: §L.2 Proposal 流程只说"必须先写 proposal"，但没说"候选超过 N 条或超过 M 天要批量升格"。

#### F-W20-02: silent merge 反模式 — 规则改了但 proposal 没立

- **现象**: 至少 3 处规则变更已落地但 proposal 文件未创建:
  - `.claude/rules.md §D #5-7` (编码硬规则 / curl --data-binary / EFBFBD 检测) → 对应 signals 候选 0028/0029
  - `plm-backend/scripts/new-business-module.sh` (业务模块生成器) → 对应 signals 候选 0027/0034
  - 这些是周六编码事故 + v0.4 启动时"用户明确请求绕过 proposal 直接改"，但**事后没有补录** proposal 文件
- **影响**: 违反 §L.2 例外条款 "User-requested-bypass 必须事后补录"；未来回看历史会看到规则变了但找不到决策依据。
- **根因**: 当时 focus 在事故根治，事后 cleanup 遗漏；缺少"周末统一补录" 仪式。

#### F-W20-03: Phase 05/06 模板对 `solo + internal-tool + early` 三条件叠加 friction 暴露

- **现象**: Phase 05 §H (双人签字) / §B.1 (上线 Checklist) / §C (凭据红线) 与 Phase 06 §B (5 指标看板) / §G (OKR 进度) / cycle 启动+终态签字 共 6 处 friction，全在 signals 候选 0007-0012。

#### F-W20-04: dogfood Phase 02 暴露 "设计文档级"约束盲区

## 2. 诊断（Diagnoses）

### 2.1 主诊断: 反思 → proposal 通道堵塞，治理债务积累

W20 是"高产周" — 37 commit、25+ Gate 实例、6 proposal merged、28 候选堆积。但**生产 friction 信号的速度 (28/周) 远超处理 proposal 的速度 (6/周)**，结果就是治理债务积累，下周如果没批量泄洪，月底候选会突破 50 条。

诊断:
- **第一性原理**: 自进化机制的健康度 = 信号生产速度 / 信号处理速度。当前比值是 28/6 ≈ 4.7 → 不可持续。
- **缓解路径**: 周末固定"批量升格"仪式 (本份 reflect 的核心任务) + Phase B `/reflect-weekly` 自动化候选筛选。

### 2.2 次诊断: Phase 05/06 模板债务

Phase 01-04 已经升级到 4 维参数化 (L级 × 项目类型 × 团队规模 × 成熟度)，但 Phase 05/06 还停在 v1。每个跑到 Phase 05/06 的模块都会暴露同样的 friction → 应该批量升级而不是逐次豁免。

### 2.3 第三诊断: silent merge 是 §L.2 例外条款被滥用

§L.2 设计意图是"事故/紧急情况可以绕过 proposal 直接改，但事后必须补录"。W20 周六的编码事故 + 业务模块生成器都属于此类，但**补录环节遗漏**。这次周末闭合反思补这块债。

## 4. 元复盘 — 关于"本份周报本身"

本份 W20 周报与之前两份 ad-hoc dogfood reflect 的区别:

| 维度 | dogfood reflect (事件触发) | W20 周报 (周末闭合) |
|---|---|---|
| 触发 | 走完一个 Gate | 周末固定时间 |
| 输入 | 1 个 Gate 实例 + 1-3 个相关 commit | 整周 git log + 全部 Gate 实例 + signals 候选堆积 |
| 输出深度 | 找 3-10 个具体 friction | 找系统性模式 + 升格候选 |
| 提案产出节奏 | 每份 ad-hoc 产 3 个 proposal | 每周末批量产 5-10 个 proposal |

→ **dogfood reflect 是"显微镜"，周报是"望远镜"**，两者互补。
