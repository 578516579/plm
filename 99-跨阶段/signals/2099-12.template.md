## 8. 测试编排（Test Orchestration）

> 由 `test-orchestrator` 每轮测试收口时记录（rules §G.5.4 MUST）。**flake 与真退步必须分开**。

| 字段 | 当月 | 说明 |
|---|---|---|
| e2e_flake_count | | 复测(`--retries=1`)后转绿的"假失败"次数 |
| e2e_real_fail_count | | 复测仍红的"真退步"次数 |
| coverage_gap | | 发现的覆盖缺口(哪层/哪模块) |
| rca_category | | 失败根因分布:env / schema / stale-jvm / code / contract / encoding |
| encoding_guard_breaks | | encoding 守门红的次数(应=0,P0) |
| gate_evidence_backfill_attempt | | 贴历史证据糊弄 Gate 的企图(应=0) |

> **触发提案条件**：同类 flake 月内 ≥3 → 测试稳定性提案；coverage_gap 反复同一层 → 补该层模板提案；encoding_guard_breaks > 0 → P0 复盘；gate_evidence_backfill_attempt > 0 → 严重违规调查。
