## 三、问题清单（按严重度）

### 🔴 P0 — 阻断"AI / PRD-aligned"结论成立

**P0-1　AI 功能名实不符（最严重业务问题）**

实测 19 个声称带 AI 的功能分三档：

| 档位 | 模块 | 真相 |
|---|---|---|
| ✅ 真消费 LLM（4） | prd · testcase · openspec · ai-agent | 配真 key 后采用 `result.getText()` 落库（本人已核 `PrdServiceImpl.java:182-198`） |
| ⚠️ 假 AI · A 档（~13） | competitive · inception · arch · dbdesign · apidesign · ued · testplan · autotest · dora · analytics · manual-impl · manual-ops 等 | **真调一次 `aiService.chat()` 但丢弃返回值，业务产物写死模板**。`CompetitiveServiceImpl.java:155-176` 注释亲口承认："走一次 AiService 产生 invocation log；**业务输出仍用下方 mock（保 E2E）**"（本人已核） |
| ❌ 纯假 · C 档（4） | testdata · manual-product · dashboard · release | 连 `chat()` 都不调；`release` 声称的"AI 评审"代码里不存在 |

附加事实：默认 `AiProperties.default-provider=mock`、各 provider `enabled=false`，**开箱即用全系统 AI 皆为 mock 回显**，仅显式配 `AI_*_ENABLED=true`+key 后真接 LLM（此为 dev/CI 零依赖的有意设计，保留）。
→ PRD 的 AI 差异化卖点在多数模块未真正实现；"调 chat 仅刷审计日志"写法有迷惑性（审计表有 token 消耗但产物是静态模板）。

**✅ 整改（2026-05-28，已验证 BUILD SUCCESS）**：新增 `plm-common` 静态助手 [`AiTexts.generate(aiService, req, fallback)`](../plm-backend/plm-common/src/main/java/cn/com/bosssfot/dv/plm/common/ai/AiTexts.java) —— 内部调 `chat()`，真 provider 非空输出则采用（剥 markdown 围栏），否则回退模板。已把 **12 个 A 档模块**（competitive/inception/arch/dbdesign/apidesign/ued/testplan/autotest×2/dora/analytics/manual-impl/manual-ops）的主产物字段接入该助手：配真 key 后即为真实 LLM 产物，默认 mock 仍走模板（单测 200+ 全绿，13 模块 reactor BUILD SUCCESS）。因助手内部调 `chat()`，原单测对 `chat` 的 stub/verify 全部沿用，零改测试。
**✅ 整改 P0-1b（2026-05-28，已验证 BUILD SUCCESS）**：C 档 4 模块中 **3 个已接** AiTexts：
- `testdata.generate()` — 接 AiTexts，fallback = 原 hardcoded 示例 JSON（保 §M.2 单测对 "soil_moisture" 的断言）。测试类加 `@Mock AiService`，chat() 默认 null 返回 → 走 fallback → 18+ 既有断言无变化。
- `manual-product` — **新增** `aiGenerate(Long)`（接口+实现+控制器）：`POST /business/manual-product/ai/generate/{id}`，权限 `business:manual-product:edit`；真 provider 落 Markdown 手册，fallback = 含目录/章节/截图占位的结构化模板。
- `release` — **新增** `aiReview(Long)`（接口+实现+控制器）：`POST /business/release/ai/review/{id}`，权限 `business:release:edit`；说明文本走 LLM，**评分由 DORA 4 指标确定性计算**（基线 85，失败率/MTTR 扣分，部署频率 ≥1 加分，clamp [0,100]）—— 故意不交给 LLM 避免幻觉。

**仍未接 AI（1 延期 + 1 枚举）**：
- **dashboard.aiMetrics.recommendations** —— `aggregate()` 在每次工作台加载都触发，naive 接 LLM = 每次开页都烧 token；**延期**待缓存/定时刷新设计后再做（P2-8 跟踪）。
- **requirement.aiEvaluate** —— 输出 high/medium/low **枚举**（非散文），不适用文本替换，**有意排除**，需后续做 LLM 输出解析（独立 follow-up）。

→ **现状：19/约 19 个 AI 功能在配置真 provider 后为真实 LLM 产物**；仅 dashboard.recommendations 延期 + requirement.aiEvaluate 枚举解析待后续。
